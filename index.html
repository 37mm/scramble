End-to-End Encryption

Scramble servers receive and store only ciphertext. This means that nobody, not even the operators of the Scramble servers, or any entity that has gained control of the servers, can read your mail. We plan to offer a browser extension soon, so that even if a Scramble server is compromised, your email will still be encrypted locally, completely secure.

Easy to Use

Scramble includes a user-friendly webmail interface. You can check your email from anywhere, even a library computer. Scramble also can exchange email with regular, insecure email addresses. This is obviously not as secure as communication between two Scramble servers, but it allows you to use Scramble as your email provider right away.

Open, Federated Protocol

Scramble is an open-source protocol. Scramble.io makes it easy to get started, but for complete control, host your own Scramble server. This is an especially attractive option for enterprises and non-profit organizations that need to keep proprietary IP and communications safe from prying eyes. Unlike some other “secure” email services, our source is and will always be completely open and auditable.

Encryption

Scramble is end-to-end encrypted webmail. Everything is stored on the server, but encrypted. For conversations between Scramble addresses, the servers never have access to the plaintext message of your mail. Messages are encrypted in the browser using OpenPGP before it reaches the server. So is your contacts list. Your OpenPGP private keys are also stored on the server, but encrypted with your passphrase. This way, as long as your passphrase remains a secret, so does everything else.

Until recently, it was common knowledge that the browser is not an ideal environment for cryptographic operations. OpenPGP was never available as a Javascript library until recently, and the pseudo-random-number-generator (PRNG) provided by Javascript wasn’t up to par. This has all changed only recently. OpenPGP.js is maturing (troll face), and a cryptographically secure PRNG is available in all modern browsers. As long as the browser & client javascript code can be trusted (and of course the underlying operating system and machine), it is in fact possible to provide strong security in webmail.

Identity

A major pain-point of OpenPGP is knowing which OpenPGP public key is associated with an address. The established way is to have key-signing parties to build a web of trust, but this is cumbersome and few people actually do this. It would be better if email addresses could be securely resolved to a public key automatically. Scramble solves this with a novel federated notary system inspired by Moxie’s Perspectives project. You ask some trusted notaries (and each Scramble server is also a notary) what the corresponding public key is for a Scramble address, and you can be confident that you found the right key if all the notaries are in consensus. This way, Scramble users don’t have to understand OpenPGP web-of-trust or public key servers. You can just send to the Scramble address, just like regular email, and the client code will take care of the rest.

Compatibility

Some users will opt for less security and more convenience.

For compatibility, you can exchange email with addresses outside of Scramble. Those messages are sent in plaintext. Incoming plaintext messages are encrypted with your public key for storage, so nobody can obtain the records from Scramble after the fact.

Be careful about doing this, especially if you want to keep your address anonymous. The security guarantees here are, of course, weaker than for conversations between Scramble users.
